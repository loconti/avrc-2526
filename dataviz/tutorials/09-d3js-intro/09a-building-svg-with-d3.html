09a — Building SVG with D3

Before any data, before any scales, learn the mechanics: how a D3 page is wired and how to draw an SVG by hand using d3.select, .append, .attr, and .style. The middle of this tutorial walks through every SVG primitive and the attributes you can customise on each.

1. The HTML scaffolding

A D3 page is a normal HTML page with two extra ingredients: the D3 library, loaded from a CDN, and a container element where the chart will live. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My chart</title>
    <script src="https://d3js.org/d3.v7.min.js"></script>
</head>
<body>
    <h1>My chart</h1>
    <div id="chart"></div>
    <script>
        // D3 code goes here. The variable d3 is now global.
    </script>
</body>
</html>
  • the <script src="…d3.v7.min.js"> tag loads the D3 library and exposes d3 as a global
  • the <div id="chart"> is an empty container we will append SVG into
  • our own <script> tag runs after the body, so the container exists by the time D3 looks it up

2. The first D3 line

D3's central idea is method chaining: every method returns a selection, so you can keep calling methods on the same object. The simplest possible D3 line: 

d3.select("#hello-target")
    .append("p")
    .text("Hello from D3!");

Read it as: select the element with id hello-target, append a <p>, set its text content. Click the button to run it (it appends a new paragraph each click):

3. Building an SVG canvas

An SVG canvas is just an SVG element with a width and a height. D3 builds it the same way as a paragraph — select a container, append, set attributes: 

const svg = d3.select("#canvas-target")
    .append("svg")
        .attr("width",  300)
        .attr("height", 150)
        .style("background", "#fafafa")
        .style("border",     "1px solid #ddd");

Notice that append("svg") returns a new selection wrapping the freshly created <svg> element, so the subsequent .attr and .style calls operate on it, not on the parent div.

The next six sections walk through every SVG primitive — <rect>, <circle>, <line>, <path>, <text>, and <g> — with several variants per element and the attributes you can change.

4. <rect> — rectangles

The simplest mark: an axis-aligned rectangle. Position by its top-left corner (x, y) and size by width and height.

// basic
svg.append("rect")
    .attr("x", 30).attr("y", 30)
    .attr("width", 80).attr("height", 60)
    .attr("fill", "#0072B2");

// thick stroke
svg.append("rect")
    .attr("x", 150).attr("y", 30)
    .attr("width", 80).attr("height", 60)
    .attr("fill",   "#0072B2")
    .attr("stroke", "#1a1a1a")
    .attr("stroke-width", 4);

// dashed stroke (no fill)
svg.append("rect")
    .attr("x", 270).attr("y", 30)
    .attr("width", 80).attr("height", 60)
    .attr("fill", "none")
    .attr("stroke", "#0072B2")
    .attr("stroke-width", 2)
    .attr("stroke-dasharray", "6,4");

// rounded corners (rx, ry)
svg.append("rect")
    .attr("x", 390).attr("y", 30)
    .attr("width", 80).attr("height", 60)
    .attr("rx", 12).attr("ry", 12)
    .attr("fill", "#0072B2");

// semi-transparent
svg.append("rect")
    .attr("x", 510).attr("y", 30)
    .attr("width", 80).attr("height", 60)
    .attr("fill", "#0072B2")
    .attr("opacity", 0.35);
AttributeMeaning
x, ytop-left corner of the rectangle
width, heightsize in pixels
fillinterior color (use "none" for an outline-only rect)
stroke, stroke-widthoutline color and thickness
stroke-dasharraydash pattern, e.g. "6,4" = 6px on, 4px off
rx, rycorner radius (rounded rectangles)
opacity, fill-opacityoverall / fill-only transparency, 0–1

5. <circle> — circles

Centred at (cx, cy) with radius r. The most common mark in scatterplots and bubble charts.

// basic
svg.append("circle")
    .attr("cx", 60).attr("cy", 60).attr("r", 28)
    .attr("fill", "#0072B2");

// ring (no fill, just stroke)
svg.append("circle")
    .attr("cx", 180).attr("cy", 60).attr("r", 28)
    .attr("fill", "none")
    .attr("stroke", "#0072B2").attr("stroke-width", 3);

// row of varying radii — same cy, increasing r
[8, 14, 20, 26, 32].forEach((r, i) => {
    svg.append("circle")
        .attr("cx", 320 + i * 56).attr("cy", 60).attr("r", r)
        .attr("fill", "#0072B2").attr("opacity", 0.7);
});
AttributeMeaning
cx, cycentre of the circle
rradius in pixels
fillinterior color ("none" → ring)
stroke, stroke-widthoutline color and thickness
opacity0–1; use ~0.6–0.8 when many circles overlap

6. <line> — line segments

A straight segment from (x1, y1) to (x2, y2). Lines have no fill — only stroke attributes apply.

// thin solid
svg.append("line")
    .attr("x1", 30).attr("y1", 60)
    .attr("x2", 130).attr("y2", 60)
    .attr("stroke", "#1a1a1a");

// thick solid
svg.append("line")
    .attr("x1", 170).attr("y1", 60)
    .attr("x2", 270).attr("y2", 60)
    .attr("stroke", "#1a1a1a")
    .attr("stroke-width", 6);

// dashed
svg.append("line")
    .attr("x1", 310).attr("y1", 60)
    .attr("x2", 410).attr("y2", 60)
    .attr("stroke", "#1a1a1a")
    .attr("stroke-width", 2)
    .attr("stroke-dasharray", "6,4");

// thick with rounded caps
svg.append("line")
    .attr("x1", 450).attr("y1", 60)
    .attr("x2", 550).attr("y2", 60)
    .attr("stroke", "#0072B2")
    .attr("stroke-width", 10)
    .attr("stroke-linecap", "round");
AttributeMeaning
x1, y1, x2, y2the two endpoints
stroke, stroke-widthcolour and thickness
stroke-dasharraydash pattern (e.g. "6,4", "2,3" for dotted)
stroke-linecapline ends: "butt" (default), "round", "square"

7. <path> — arbitrary shapes

The most flexible primitive. The shape is described by the d attribute, a tiny string language: M x y (MoveTo), L x y (LineTo), Q cx cy x y (Quadratic curve), C c1x c1y c2x c2y x y (Cubic), Z (close back to the start). In real D3 work we rarely write these strings by hand — there are shape generators (covered in DV10) that build them for us. Here we write a few by hand to make the language concrete:

// triangle: move to corner, lines to the other two, close
svg.append("path")
    .attr("d", "M 60 110 L 100 30 L 140 110 Z")
    .attr("fill", "#0072B2");

// arrow (closed shape)
svg.append("path")
    .attr("d", "M 220 50 L 280 50 L 280 30 L 320 70 L 280 110 L 280 90 L 220 90 Z")
    .attr("fill", "#E69F00");

// smooth curve (Q = quadratic Bezier)
svg.append("path")
    .attr("d", "M 380 90 Q 460 10, 540 90")
    .attr("fill", "none")
    .attr("stroke", "#009E73").attr("stroke-width", 3);
AttributeMeaning
dthe path data — a sequence of commands and coordinates
fillinterior; use "none" for an open curve
stroke, stroke-widthoutline of the path
Most common d commands: M x y move to · L x y line to · Q cx cy x y quadratic curve · C c1x c1y c2x c2y x y cubic curve · A … arc · Z close. Lower-case (m, l, …) means relative coordinates.

8. <text> — labels

Position the text's baseline at (x, y). The horizontal relationship of the text to that point is set by text-anchor:

// the same x for all three; only text-anchor changes
svg.append("text").attr("x", 150).attr("y", 50)
    .attr("text-anchor", "start") .text("anchor=start");

svg.append("text").attr("x", 350).attr("y", 50)
    .attr("text-anchor", "middle").text("anchor=middle");

svg.append("text").attr("x", 550).attr("y", 50)
    .attr("text-anchor", "end")   .text("anchor=end");

// styling: size, weight, stroke
svg.append("text").attr("x", 30).attr("y", 110)
    .style("font-size", "11px").attr("fill", "#666").text("small");

svg.append("text").attr("x", 130).attr("y", 110)
    .style("font-weight", 700).text("BOLD");

svg.append("text").attr("x", 240).attr("y", 110)
    .attr("fill", "#0072B2").attr("stroke", "#1a1a1a").attr("stroke-width", 0.5)
    .text("stroked");
AttributeMeaning
x, yposition of the text's baseline anchor point
dx, dysmall offsets from (x, y) — handy for fine-tuning
text-anchor"start" | "middle" | "end" — horizontal alignment relative to (x, y)
font-sizesize in pixels (use style with "px")
font-family, font-weighttypeface and weight
fill, stroketext color; stroke is rare but useful for outline effects

9. <g> — groups and transforms

A <g> draws nothing of its own; it groups child elements so that a single transform applies to all of them. The transform composes — translate, rotate, and scale can stack.

// build a tiny "house" inside a group, then place it three different ways
function house(parent) {
    parent.append("rect")
        .attr("x", -20).attr("y", -10).attr("width", 40).attr("height", 30)
        .attr("fill", "#0072B2");
    parent.append("path")
        .attr("d", "M -22 -10 L 0 -32 L 22 -10 Z")
        .attr("fill", "#E69F00");
}

// just translated
const a = svg.append("g").attr("transform", "translate(80, 70)");
house(a);

// translate + rotate
const b = svg.append("g").attr("transform", "translate(220, 70) rotate(20)");
house(b);

// translate + scale
const c = svg.append("g").attr("transform", "translate(360, 70) scale(1.4)");
house(c);

// nested: inner translate adds to outer
const outer = svg.append("g").attr("transform", "translate(480, 50)");
const inner = outer.append("g").attr("transform", "translate(40, 20) rotate(-15)");
house(inner);
TransformWhat it does
translate(x, y)shift children by (x, y)
rotate(deg)rotate children around (0, 0); use rotate(deg cx cy) for a different pivot
scale(s)scale uniformly; scale(sx, sy) for non-uniform
Multiple transforms separated by spaces apply right-to-left: in translate(…) rotate(…) the rotation happens first, then the translation. Nested <g>s achieve the same composition more explicitly.

10. The SVG coordinate system

Two facts that catch every newcomer:

  • the origin (0, 0) is at the top-left corner of the SVG
  • y grows downward — the opposite of math notation

11. Live attribute playground

Move the sliders to change the attributes of a single circle. Each slider is wired with a plain DOM event listener that reaches into the SVG via d3.select and updates one attribute. Same mechanic as a fully interactive chart, just on a single element.

// One circle, kept in a variable so we can update it.
const dot = d3.select("#playground").append("svg")
    .attr("width", 540).attr("height", 300)
  .append("circle")
    .attr("cx", 240).attr("cy", 120).attr("r", 24)
    .attr("fill", "#0072B2");

// On every slider input, set one attribute on the circle.
d3.select("#slider-cx").on("input", function () { dot.attr("cx", +this.value); });
d3.select("#slider-cy").on("input", function () { dot.attr("cy", +this.value); });
d3.select("#slider-r" ).on("input", function () { dot.attr("r",  +this.value); });
Takeaway. Every D3 chart is, at the bottom, a chain of select → append → attr → style calls operating on SVG elements. The next tutorial (09b) replaces hand-appended elements with a data-driven join: one DOM element per row of an array.
