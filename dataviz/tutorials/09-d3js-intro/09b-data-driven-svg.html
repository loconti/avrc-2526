09b — Data-Driven SVG

First contact with the D3 data join: from an array of objects to one SVG element per row. We use a small toy dataset and place circles at literal pixel positions — no scales, no axes — so the binding between data and DOM stays in plain sight.

1. The dataset

A small array of nine plain JavaScript objects. Each row has an x, a y, and a value. Coordinates are in pixels of the plot area — there is no domain conversion yet (scales and axes are introduced in 09c).

const dataset = [
    { x: 100, y:  80, value: 20 },
    { x: 200, y: 150, value: 15 },
    { x: 300, y: 100, value: 25 },
    { x: 400, y: 200, value: 10 },
    { x: 500, y:  80, value: 30 },
    { x: 150, y: 250, value: 18 },
    { x: 250, y: 180, value: 12 },
    { x: 350, y: 150, value: 22 },
    { x: 450, y: 120, value: 16 }
];

2. The data join

The data join binds the array of objects to a selection of DOM elements. For each row it creates a <circle> and lets us set attributes from the bound row.

svg.selectAll("circle")
    .data(dataset)
    .join("circle")
        .attr("cx",   d => d.x)
        .attr("cy",   d => d.y)
        .attr("r",    d => d.value)
        .attr("fill", "steelblue")
        .attr("opacity", 0.7);

Read the four lines as four steps:

  1. selectAll("circle") — start with a selection of all circles inside the SVG (initially empty)
  2. .data(dataset) — bind the array to that selection
  3. .join("circle") — make the DOM look like the data: create a <circle> for every row that does not yet have one
  4. .attr(name, d => ...) — set each attribute as a function of the bound row d; literal values can also be passed (no function)

That is the entire workflow. Every D3 chart we build from now on follows the same shape: select → bind data → join → set attributes from the data.

3. The three sub-selections: enter, update, exit

The interesting case is when the data changes. Calling the join a second time with a different array partitions the work into three sub-selections:

  • enter — data rows that have no matching DOM element yet, so D3 creates one
  • update — data rows that already match an existing element, so D3 keeps it (any attributes you set will be refreshed)
  • exit — DOM elements whose data has disappeared, so D3 removes them

Two datasets, matched by id:

  • dataset A: ids 1, 2, 3, 4, 5, 6, 7, 8, 9
  • dataset B: ids 3, 4, 5, 6, 7, 8, 9, 10, 11

Going from A to B, ids 1, 2 exit; ids 3–9 update; ids 10, 11 enter. The three rows below all share the same id axis (1 on the left, 11 on the right) so the same id always sits at the same horizontal position. A circle is drawn only where its id belongs to that sub-selection.

currently showing the join going A → B

"Going A → B" means: start with dataset A on screen, then call the join with dataset B — the diagram shows what the join did. Click the button to flip the direction (B → A swaps the enter and exit roles).

Each circle is labelled with its id. The story to read off the diagram: "this is what the join just did" — green circles were created, blue circles were kept, orange circles were removed.

4. The reactive reshuffle

The same code, run on a constantly-changing dataset. Each click of the button below replaces roughly half the points: D3 adds the new ones, removes the old ones, and keeps the rest. The transitions on enter and exit make the three sub-selections visible as motion.

The join can take three callbacks — one per sub-selection — so we can give each its own behaviour. Here, entering circles fade in by growing from r = 0; exiting circles shrink to r = 0 before being removed; updates are left untouched.

// `points` is the array of currently-shown rows; each row has a stable `id`.
let points = makePoints(20);

function render() {
    plot.selectAll("circle")
        .data(points, d => d.id)            // key by id, so D3 matches rows to circles
        .join(
            enter => enter.append("circle")
                .attr("cx", d => d.x).attr("cy", d => d.y)
                .attr("r", 0)                                      // start invisible
                .attr("fill", "steelblue").attr("opacity", 0.7)
                .call(en => en.transition().duration(400)
                    .attr("r", d => d.value)),                 // grow into place

            update => update,                                   // keep as-is

            exit  => exit.call(ex => ex.transition().duration(300)
                    .attr("r", 0).remove())                        // shrink, then remove
        );
}
render();

// On every click, drop ~half the points and add fresh ones, then re-render.
document.getElementById("reshuffle").addEventListener("click", () => {
    const keep = points.filter(() => Math.random() > 0.5);
    points = keep.concat(makePoints(20 - keep.length));
    render();
});

Takeaway. A D3 chart is a function from data to a set of DOM elements with attributes. The join keeps the DOM in sync with the data on every update. So far the data has been in pixel coordinates; 09c introduces scales — the functions that let real data values drive the same join.
