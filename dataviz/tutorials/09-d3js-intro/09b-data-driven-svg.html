09b — Data-Driven SVG

First contact with the D3 data join: from an array of objects to one SVG element per row. We use a small toy dataset and place circles at literal pixel positions — no scales, no axes — so the binding between data and DOM stays in plain sight.

How this tutorial is organised

Three short blocks:

  1. The toy dataset (§1). Nine objects with hand-picked pixel coordinates, so you can read off which datum maps to which circle.
  2. The data join (§2). The selectAll · data · join("tag") chain, then a line-by-line read.
  3. What .join does internally (§3). The enter / update / exit decomposition, named here as concept; the dynamic side of the same machinery (re-binding, key functions) is the topic of 10a.

1. The dataset

A small array of nine plain JavaScript objects. Each row has an x, a y, and a value. Coordinates are in pixels of the plot area — there is no domain conversion yet (scales and axes are introduced in 09c).

const dataset = [
    { x: 100, y:  80, value: 20 },
    { x: 200, y: 150, value: 15 },
    { x: 300, y: 100, value: 25 },
    { x: 400, y: 200, value: 10 },
    { x: 500, y:  80, value: 30 },
    { x: 150, y: 250, value: 18 },
    { x: 250, y: 180, value: 12 },
    { x: 350, y: 150, value: 22 },
    { x: 450, y: 120, value: 16 }
];

2. The data join

The data join binds the array of objects to a selection of DOM elements. For each row it creates a <circle> and lets us set attributes from the bound row.

svg.selectAll("circle")
    .data(dataset)
    .join("circle")
        .attr("cx",   d => d.x)
        .attr("cy",   d => d.y)
        .attr("r",    d => d.value)
        .attr("fill", "steelblue")
        .attr("opacity", 0.7);

Read the four lines as four steps:

  1. selectAll("circle") — start with a selection of all circles inside the SVG (initially empty)
  2. .data(dataset) — bind the array to that selection
  3. .join("circle") — make the DOM look like the data: create a <circle> for every row that does not yet have one
  4. .attr(name, d => ...) — set each attribute as a function of the bound row d; literal values can also be passed (no function)

That is the entire workflow. Every D3 chart we build from now on follows the same shape: select → bind data → join → set attributes from the data.

3. What .join("circle") does internally

Under the hood the join partitions its work into three named sub-selections — enter (rows with no matching element, D3 creates them), update (rows that already have an element, D3 keeps it), exit (elements whose data has disappeared, D3 removes them). For a one-shot chart only the enter case matters and the shortcut .join("circle") handles it transparently.

The three-way split becomes important the moment the data changes — re-binding with a different array, sorting the bars, scrubbing a slider. Tutorial 10a picks that story up: keys, custom enter/update/exit callbacks, and animated rebinds. For now: one bind → one chart.

Takeaway. A D3 chart is a function from data to a set of DOM elements with attributes. .join("tag") binds an array to the DOM in one line. So far the data has been in pixel coordinates; 09c introduces scales and the shape generators (d3.line, d3.area) that turn real data values into pixels.
