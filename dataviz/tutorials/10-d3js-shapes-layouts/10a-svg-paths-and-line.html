10a — SVG Paths and the Line / Area Generators

A single <path> can draw any 2D shape — straight or curved, open or closed. Writing the path string by hand is tedious; D3 ships path generators (d3.line, d3.area) that turn an array of data into a valid d attribute. This is what powers every line chart, area chart, and curved shape in D3.

1. The SVG path mini-language

An SVG <path> stores its geometry in the d attribute — a string of commands and coordinates:

<path d="M 20 120 L 100 40 L 180 90" />

The most common commands:

CommandMeaning
M x yMoveTo — lift the pen to (x, y)
L x yLineTo — draw a line to (x, y)
Q cx cy x yquadratic Bezier curve through control point (cx, cy)
C c1x c1y c2x c2y x ycubic Bezier curve through two control points
A rx ry … x yelliptical arc
Zclose the path back to the starting point

Lower-case commands (m, l, c) mean relative coordinates instead of absolute. The full path mini-language is small but powerful — every D3 generator below produces strings like the one above.

2. Reading a path string step by step

The browser executes commands in order. For M 20 120 L 100 40 L 180 90 Z:

  1. M 20 120 — pen moves to (20, 120). Nothing drawn yet.
  2. L 100 40 — line from (20, 120) to (100, 40).
  3. L 180 90 — line from (100, 40) to (180, 90).
  4. Z — line back to the start (20, 120) — closes the triangle.
// triangle, written by hand
svg.append("path")
    .attr("d", "M 20 120 L 100 40 L 180 90 Z")
    .attr("fill", "#0072B2");

3. d3.line — the line generator

For a chart, you don't write the d string by hand. d3.line() is a function builder: configure how to extract x and y from each datum, then call it on the array to produce the path string.

const line = d3.line()
    .x(d => xScale(d.year))
    .y(d => yScale(d.value));

svg.append("path")
    .datum(timeSeries)         // bind the WHOLE array to one element
    .attr("d", line)           // line(timeSeries) → "M … L … L …"
    .attr("fill", "none")
    .attr("stroke", "#0072B2")
    .attr("stroke-width", 2);

4. .datum() versus .data()

This distinction trips up everybody once.

  • .data(array) binds each row of the array to one element in the selection. Used for scatterplots, bar charts — one mark per row.
  • .datum(value) binds one whole value (often the entire array) to one element. Used for line charts and area charts — one path that consumes many rows.
// scatterplot: one circle per row
svg.selectAll("circle")
    .data(rows)               // ← data, plural
    .join("circle")
    ...

// line chart: one path consuming the whole array
svg.append("path")
    .datum(rows)              // ← datum, singular
    .attr("d", line);

5. Curves are design choices

By default d3.line connects points with straight segments. For smoother shapes, configure a .curve(...):

const line = d3.line()
    .x(d => x(d.year))
    .y(d => y(d.value))
    .curve(d3.curveMonotoneX);   // ← here

Common curves, each with a distinct visual character:

  • curveLinear — straight segments (default)
  • curveStep — horizontal-then-vertical step changes
  • curveStepAfter, curveStepBefore — variants of step
  • curveMonotoneX — smooth, restrained, never overshoots vertically
  • curveCatmullRom — smoother, more decorative
  • curveBasis — heavily smoothed, may not pass through points

6. d3.area — the area generator

An area chart is also one <path>, but the path needs both an upper and a lower boundary. d3.area() takes x, y0 (lower edge), and y1 (upper edge):

const area = d3.area()
    .x(d => xScale(d.year))
    .y0(innerH)                // baseline = bottom of the plot
    .y1(d => yScale(d.value));

svg.append("path")
    .datum(timeSeries)
    .attr("d", area)
    .attr("fill", "#0072B2")
    .attr("opacity", 0.3);

Same data, same generator pattern as d3.line — only the configuration differs. Both can take a .curve(...).

Takeaway. Path generators are the bridge between numeric data and SVG path strings. The same five-method pattern (const gen = d3.line(); gen.x(); gen.y(); path.datum(data); path.attr("d", gen)) covers lines, areas, arcs, and any custom shape the chart needs.
