10b — Bar Charts and d3.stack

Bar charts look simple but force you to confront the inverted SVG y axis. Stacked bars then add a second twist: the data has to be transformed into intervals before you can draw it. d3.stack() is the helper that does that transformation, and it is the first example of a D3 layout — a function from data to drawable geometry.

1. A bar is a <rect>

For a vertical bar chart, each <rect> needs four attributes:

  • x — left edge of the bar
  • y — top edge of the bar (not the bottom!)
  • width — bar width
  • height — bar height

Because SVG y grows downward, the top of a bar has the smaller y. The height is then computed as the distance from the top down to the baseline. Most bar-chart bugs in D3 trace back to this single fact.

2. Bar chart construction, step by step

Mean life expectancy per continent for 2007 — six values, six bars. Three steps:

// step 1: categorical x-scale (one slot per continent)
const x = d3.scaleBand()
    .domain(data.map(d => d.continent))
    .range([0, innerW])
    .padding(0.15);

// step 2: linear y-scale (inverted, so larger values appear higher)
const y = d3.scaleLinear()
    .domain([0, d3.max(data, d => d.value)]).nice()
    .range([innerH, 0]);

// step 3: one rect per row
svg.selectAll("rect")
    .data(data)
    .join("rect")
        .attr("x",      d => x(d.continent))
        .attr("y",      d => y(d.value))            // top of the bar
        .attr("width",  x.bandwidth())
        .attr("height", d => innerH - y(d.value));  // top → baseline

The height formula innerH - y(d.value) grows the bar upward from the baseline. If you forget it and write height = y(d.value) instead, the bars hang from the top of the chart. (Try it locally — the result is unmistakable.)

3. scaleBand in detail

scaleBand turns discrete categories into equal-width slots. It exposes three things you need:

  • x("Asia") — left edge of the "Asia" slot, in pixels
  • x.bandwidth() — slot width minus the configured padding
  • x.step() — distance from one slot's start to the next
const x = d3.scaleBand()
    .domain(["Mon","Tue","Wed","Thu","Fri","Sat","Sun"])
    .range([0, 700])
    .padding(0.15);     // 0 = bars touch  ·  1 = bars vanish

x("Wed");            // → 270 (left edge of Wed)
x.bandwidth();       // → 80  (width of each bar)

Below: the same seven categories, three different paddings (0.05, 0.25, 0.5). Padding controls the gap between bars; pick a value that lets the bars dominate without crowding.

4. Why stacked bars need transformed data

A stacked bar chart shows multiple categories summed at each x-position. For each segment of each bar, you need:

  • the x-category (which bar)
  • the lower y of the segment (where it starts)
  • the upper y of the segment (where it ends)
  • the series key (which colour)

You can't read those intervals directly off the input data — you have to compute the running totals. d3.stack() is the helper that does it.

5. d3.stack() transforms columns into layers

Input: an array of "wide" rows where each row is one x-position and each column is one series.

const data = [
    { year: 2018, apples: 10, bananas: 20, cherries:  5 },
    { year: 2019, apples: 12, bananas: 18, cherries:  8 },
    { year: 2020, apples: 15, bananas: 22, cherries:  6 },
    { year: 2021, apples: 17, bananas: 24, cherries: 10 },
    { year: 2022, apples: 14, bananas: 21, cherries: 12 }
];

const stack = d3.stack().keys(["apples", "bananas", "cherries"]);
const series = stack(data);

Output: an array of layers, one per key. Each layer has one entry per x-position, and each entry is a [y0, y1] pair (the bottom and top of the segment). The original row is attached as .data:

// series[0]  → the "apples" layer
// series[0][0] → [0, 10]   for year 2018  (apples stacked first, so y0 = 0)
// series[1][0] → [10, 30]  for year 2018  (bananas on top of apples)
// series[2][0] → [30, 35]  for year 2018  (cherries on top of bananas)
// series[0][0].data → { year: 2018, apples: 10, bananas: 20, cherries: 5 }

6. Drawing a stacked bar chart

The shape of the output (an array of arrays) maps directly to a nested selection: one <g> per layer, one <rect> per segment inside. 

svg.selectAll("g.layer")
    .data(series)
    .join("g")
        .attr("class", "layer")
        .attr("fill", d => color(d.key))
  .selectAll("rect")
    .data(d => d)                          // children = segments of THIS layer
    .join("rect")
        .attr("x",      d => x(d.data.year))
        .attr("y",      d => y(d[1]))      // top of the segment
        .attr("height", d => y(d[0]) - y(d[1]))  // top - bottom
        .attr("width",  x.bandwidth());

The y-domain comes from the maximum upper edge across all layers, not from any single column:

const yMax = d3.max(series, layer => d3.max(layer, d => d[1]));
const y = d3.scaleLinear().domain([0, yMax]).nice().range([innerH, 0]);

7. d3.stack is one example of a D3 layout

D3 ships several "layouts" that follow the same pattern: data goes in, drawable geometry comes out:

  • d3.stack — stacked bars / areas
  • d3.pie — angles for pie / donut slices
  • d3.hierarchy + d3.tree / d3.treemap / d3.pack — hierarchical layouts
  • d3.forceSimulation — physical simulation for force-directed graphs

Once you've built one chart with d3.stack, the rest of the layout family is approachable — the API shape is consistent.

Takeaway. A bar chart is one rectangle per row, with y meaning "top" and height = innerH − y(value). A stacked bar chart starts with a layout (d3.stack) that converts your data into per-segment [y0, y1] intervals, and a nested join (g per layer, rect per segment) draws them.
