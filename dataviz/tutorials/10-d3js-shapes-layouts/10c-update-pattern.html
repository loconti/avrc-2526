10c — Update Pattern and selection.join

Static charts answer one question for one dataset. Real visualizations change: users filter, sort, scrub a slider, switch a year. The data join is what keeps the DOM in sync with the data on every change. This tutorial covers the General Update Pattern, why key functions matter, and how selection.join() is the modern shortcut for all of it.

1. Why updates matter

Most useful charts respond to change:

  • filter the data — one continent, one year, one threshold
  • sort the bars — alphabetical or by value
  • scrub a year slider — re-render the same chart for a new snapshot
  • animate between states — transitions on enter and exit

Calling the join with the new data array does all of this. The same code that drew the chart the first time updates it on every subsequent call.

2. Recap: enter, update, exit

From 09b — the join compares the data array to the existing DOM elements and produces three sub-selections:

  • enter — data rows with no matching element yet → D3 creates one
  • update — data rows that already match an existing element → D3 keeps it
  • exit — DOM elements whose data has disappeared → D3 removes them

This three-way split is called the General Update Pattern (GUP). Modern D3 hides it behind a single call (.join("circle")), but understanding the pattern is what makes custom enter/update/exit behaviour readable.

3. The older explicit pattern

Before selection.join existed (D3 v5+), every update wrote the three sub-selections by hand. You will still see this in older examples:

// older pattern — three sub-selections, .merge() to combine enter+update
const sel = svg.selectAll("circle").data(data, d => d.id);

sel.exit().remove();                         // 1. remove old

sel.enter()                                   // 2. create new
    .append("circle")
    .attr("fill", "steelblue")
  .merge(sel)                                 // 3. enter + update together
    .attr("cx", d => x(d.value))
    .attr("cy", d => y(d.group))
    .attr("r", 5);

The pattern works but is verbose. selection.join is the modern shortcut.

4. selection.join() — the modern default

One line replaces the entire dance:

svg.selectAll("circle")
    .data(data, d => d.id)
    .join("circle")            // ← creates / keeps / removes circles to match the data
        .attr("cx", d => x(d.value))
        .attr("cy", d => y(d.group))
        .attr("r", 5)
        .attr("fill", "steelblue");

Pass "circle" (or any tag name) and D3 handles enter, update, and exit for you. Attribute calls after the join run on the merged selection — both the freshly created and the kept-around circles get them. This is the form to write first.

5. Keys preserve object identity

The second argument to .data() is the key function:

.data(data, d => d.id)

Without a key, D3 matches data to elements by index. That is fine for a one-shot chart, but breaks the moment the data is sorted, filtered, or shuffled — D3 thinks the first row always corresponds to the first DOM element, even after a sort.

The demo below has the same five items and a "shuffle" button. Switch between index join and key join to see the difference: with index join, a shuffle re-uses the existing circles in their old positions and just re-labels them; with key join, each circle keeps its identity (and its colour) when its position changes.

Try this: shuffle once with the default (key join), then switch to index join and shuffle again. Watch what happens to each circle's colour and label.

6. Custom join behaviour — three callbacks

For richer behaviour, .join accepts three callbacks — one per sub-selection:

selection.data(data, d => d.id)
    .join(
        enter  => enter.append("circle")
            .attr("r", 0)                                          // start invisible
            .attr("fill", "#009E73")                               // green for new
            .call(en => en.transition().duration(400)
                .attr("r", 6)),                                    // grow into place

        update => update
            .attr("fill", "#0072B2"),                              // blue for kept

        exit   => exit
            .attr("fill", "#D55E00")
            .call(ex => ex.transition().duration(300)
                .attr("r", 0).remove())                            // shrink and go
    );

Each callback returns a selection. D3 merges enter's return value with the update selection automatically, so subsequent .attr calls on the result of .join(...) run on both new and kept elements.

Takeaway. Every D3 chart that responds to change runs the same three steps: bind the new data with a key function, call .join, and let it create / keep / remove elements. For animated or differentiated entry/exit, pass three callbacks. The next tutorial — 10d — adds transitions, the language D3 uses to make those changes visible over time.
