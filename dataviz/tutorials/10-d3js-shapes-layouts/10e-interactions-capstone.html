10e — Interactions Capstone

A static chart answers one question. An interactive chart lets the reader ask their own. This tutorial wires up the four pieces from L10 — generators, joins, transitions — into a single Gapminder scatterplot that responds to hover, click, and a year slider. Same data flow as every previous chart, with input events feeding back into the join.

1. Listening for user actions

D3 selections forward DOM events through selection.on:

svg.selectAll("circle")
    .on("mouseover", (event, d) => {
        console.log("hover on", d.country);
    });

The callback gets two arguments:

  • event — the native browser event (with event.pageX, pageY, currentTarget, etc.)
  • d — the datum bound to the element that fired the event

This is how visual marks become interactive. The most common events:

  • mouseover · mouseout · mousemove
  • click
  • input · change (form controls)
  • keyboard events on the document or window

2. The event → state → render loop

Most interactive D3 charts follow the same three-step loop:

  1. Event — a user action fires (a hover, a click, a slider drag).
  2. State — the callback updates a small piece of state (a selected continent, a current year).
  3. Render — a single render function reads the state and re-runs the data join.
let selectedContinent = null;       // state

function render() {                  // render reads state
    const visible = selectedContinent
        ? data.filter(d => d.continent === selectedContinent)
        : data;

    svg.selectAll("circle")
        .data(visible, d => d.country)
        .join("circle")
            .attr("cx", d => x(d.gdpPercap))
            .attr("cy", d => y(d.lifeExp))
            ...;
}

legend.on("click", (event, d) => {   // event updates state
    selectedContinent = (selectedContinent === d) ? null : d;
    render();
});

This split keeps event callbacks tiny — they only touch state. All drawing logic lives in one place, which makes the chart easier to debug and extend.

3. Hover tooltip — the simplest interaction

The smallest useful interaction: a tooltip on hover. We use a single absolutely-positioned <div> as a shared tooltip rather than appending one DOM element per row.

const tip = d3.select("#tooltip");

svg.selectAll("circle")
    .on("mouseover", (event, d) => {
        tip.style("opacity", 1)
           .html(`<b>${d.country}</b><br>life exp: ${d.lifeExp.toFixed(1)} yr`);
    })
    .on("mousemove", (event) => {
        tip.style("left", (event.pageX + 12) + "px")
           .style("top",  (event.pageY + 12) + "px");
    })
    .on("mouseout", () => tip.style("opacity", 0));

4. Click to toggle — state lives in a variable

Clicking a continent in the legend filters the chart to just that continent; clicking the same continent again clears the filter. The legend item gets a visual highlight while it's the active filter:

let active = null;

legendItems.on("click", (event, d) => {
    active = (active === d) ? null : d;
    render();              // re-render with the new filter
    legendItems.attr("opacity", item => active && item !== active ? 0.3 : 1);
});

5. The year slider — scrubbing through time

Gapminder spans 1952–2007. A range slider over the years lets the reader scrub through time; every input event triggers a re-render with the new year's data:

const yearSlider = d3.select("#year");

yearSlider.on("input", function () {
    currentYear = +this.value;
    render();
});

6. Capstone: Gapminder, interactive

Everything from this tutorial — events, state, render loop — combined into one chart:

  • Hover a circle to see the country name and stats.
  • Click a continent in the legend to filter (click again to clear).
  • Drag the slider to change the year.

7. Built-in interaction helpers

For richer behaviours D3 ships dedicated helpers — they handle the messy event details so your code stays declarative:

  • d3.brush — drag a rectangular selection over the chart
  • d3.drag — make individual marks draggable (force-directed graphs use this)
  • d3.zoom — pan and zoom transforms on a chart

Each follows the event-state-render pattern: the helper fires events, your callback updates state, a render function redraws. Same shape, different inputs.

Takeaway. The three concepts in L10 (paths and generators · the update pattern · transitions) all converge in this capstone. Add a tiny event-state-render loop and you have an interactive D3 chart. From here, every chart in the dv11/dv12 unit (force layouts, chord diagrams, scrollytelling) is a richer composition of the same primitives.
